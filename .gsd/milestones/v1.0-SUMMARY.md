# Milestone: v1.0 (MVP)

## Completed: 2026-06-14

## Deliverables
- ✅ **Serverless PeerJS (WebRTC) Steuerung**: Volle P2P-Konnektivität zwischen Smartphones und Dashboard ohne zusätzlichen Server-State (abgesehen vom Standard PeerJS Cloud Signaling).
- ✅ **Rundenbasiertes Gameplay**: Klare Steuerung des aktiven Spielers vom Host aus, inkl. dynamischer Ansichten auf Client-Geräten (Warten vs. Agieren).
- ✅ **3D-CSS-Würfel-Rendering**: Performante, browsernative 3D-Simulation mit CSS Transforms und akkumulierter Rotation für kontinuierliches Rollen.
- ✅ **Lokale Historie**: Persistenter Spielverlauf via `localStorage` des Hosts zur Wiederherstellung von Spielständen und Bestenlisten.
- ✅ **PWA Offline-Support**: Vollständige Offline-Fähigkeit für Host und Client dank Service Worker Cache-First Caching und Homescreen-Installation via Web App Manifest.
- ✅ **Synthetisiertes Audio**: Dynamisches Sound-Feedback (Würfeln, Gewonnen, Verloren, Timer) erzeugt direkt im Browser per Web Audio API (keine externen Assets).

## Lessons Learned
- **Akkumulierte Rotation bei 3D-Würfeln**: Um zu verhindern, dass sich Würfel bei aufeinanderfolgenden Würfen rückwärts drehen, muss die Rotation stetig erhöht werden (`currentRotation + delta`).
- **Verbindungsabbruch vs. Ablehnung**: Beim Trennen von WebRTC-Verbindungen wegen Namensdubletten feuert das allgemeine `'close'`-Event. Ein dediziertes Flag `isDisconnecting` verhindert, dass dem Client fälschlicherweise "Verbindung verloren" anstelle von "Name bereits vergeben" angezeigt wird.
- **Service Worker Scoping**: Der Service Worker (`sw.js`) muss zwingend im Root-Verzeichnis liegen, um den gesamten Scope (inkl. index.html und controller.html) cachen und kontrollieren zu können.
- **Web Audio autoplay restriction**: Audio darf erst nach einer Interaktion des Users (z.B. Klick auf Beitritts-Button) initialisiert/abgespielt werden, um Browser-Sperren zu umgehen.
